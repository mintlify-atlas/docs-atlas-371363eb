---
title: "Initialize bench script"
description: "Automate Frappe bench initialization with the init_frappe_bench.sh script."
---

The `init_frappe_bench.sh` script initializes a Frappe bench directory using a specified branch and starts the development server.

## Full script

```bash
#!/bin/bash

# Input:
#   - Frappe branch (optional): Default to version-15
FRAPPE_BRANCH="${1:-version-15}"


# Initialize the Frappe bench
bench init frappe-bench --frappe-branch ${FRAPPE_BRANCH}

# Start the bench
cd frappe-bench
source env/bin/activate
bench start

echo "Frappe bench initialized successfully!"
```

## Script walkthrough

### Optional branch argument

The script reads the first positional argument (`$1`) as the Frappe branch name. If you do not supply one, it defaults to `version-15`. This lets you target older or newer Frappe releases without editing the script.

### Bench initialization

`bench init frappe-bench --frappe-branch <branch>` creates the `frappe-bench` directory, clones the Frappe framework at the specified branch, sets up the Python virtual environment, and installs all Python dependencies.

### Starting the bench

After initialization, the script enters the `frappe-bench` directory, activates the virtual environment, and runs `bench start` to start the development server.

## Usage

```bash
# Use default version-15
./init_frappe_bench.sh

# Use a specific version
./init_frappe_bench.sh version-14
```

<Tip>
  Choose a Frappe branch that matches your ERPNext version. For example, use `version-15` with ERPNext version 15. Mixing mismatched versions will cause dependency errors during app installation.
</Tip>
