---
title: "Reset MySQL root password"
description: "Recover access to MySQL by resetting the root password when you're locked out."
---

<Warning>
  Only follow this guide if you are locked out of MySQL and cannot authenticate as root. Bypassing authentication grants unrestricted database access, so perform these steps only on a machine you control and complete them as quickly as possible.
</Warning>

There are two approaches: using an init file (recommended) or starting MySQL in safe mode. Both methods are described below.

## Method 1: Init file

<Steps>
  <Step title="Stop MySQL">
    Stop the running MySQL service and make sure no stray `mysqld` processes remain:

    ```bash
    sudo systemctl stop mysql
    killall mysqld
    ```
  </Step>

  <Step title="Create the init file">
    Create a temporary SQL file that resets the root password. Replace `your_username` with your Linux username and `NewPassword` with your desired password:

    ```bash
    sudo nano /home/your_username/mysql-init
    ```

    Add the following line to the file:

    ```sql
    ALTER USER 'root'@'localhost' IDENTIFIED BY 'NewPassword';
    ```

    Save the file with **Ctrl+X**, **Y**, **Enter**.
  </Step>

  <Step title="Start mysqld with the init file">
    Start `mysqld` pointing at the init file, then start the MySQL service normally:

    ```bash
    sudo mysqld --init-file=/home/your_username/mysql-init &
    sudo service mysql start
    ```

    MySQL will execute the `ALTER USER` statement at startup and then continue normally.
  </Step>

  <Step title="Clean up the init file">
    Delete the init file so the plaintext password is not left on disk:

    ```bash
    sudo rm /home/your_username/mysql-init
    ```
  </Step>
</Steps>

## Method 2: Safe mode

<Steps>
  <Step title="Start MySQL in safe mode">
    Start `mysqld_safe` with grant tables and networking disabled so no passwords are required:

    ```bash
    sudo mysqld_safe --skip-grant-tables --skip-networking &
    ```
  </Step>

  <Step title="Connect as root (no password)">
    Connect to the running instance without a password:

    ```bash
    sudo mysql -u root
    ```
  </Step>

  <Step title="Update the root password">
    Select the `mysql` system database and update the authentication string directly:

    ```sql
    USE mysql;
    UPDATE user SET authentication_string=PASSWORD('new_password') WHERE User='root';
    FLUSH PRIVILEGES;
    ```

    Replace `new_password` with your chosen password.
  </Step>

  <Step title="Exit MySQL">
    ```sql
    EXIT;
    ```
  </Step>

  <Step title="Stop safe mode and restart MySQL">
    Kill the safe-mode process and start MySQL in normal mode:

    ```bash
    sudo killall mysqld_safe
    sudo systemctl start mysql
    ```

    You should now be able to log in with the new root password.
  </Step>
</Steps>
